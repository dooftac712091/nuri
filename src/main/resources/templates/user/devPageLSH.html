• AuthController (계정 로그인, 계정찾기) (3)

  • 아이디나 비밀번호 찾기의 경우, redirectAttributes.addAttribute 속성을 활용해서, 리다이렉트가 되는 페이지에 파라미터를 넘긴다. 찾기의 경우 페이지 요청과 아이디와 비밀번호를 찾았을 경우 리다이렉트되는 페이지의 메소드를 만들어야한다. 즉, 요청과 조회 두 요청에 대응할 메소드를 만든다.(오버로딩) POST 방식을 넘어온 정보를 조회한 후에 그 값을 내보내준다. 만약에 redirect 된 페이지는 Model을 통해서 해당 정보를 넘기고, model이 있을 경우에 JS는 div를 생성하여 찾은 아이디를 출력한다.

  • 1. 로그인 (HTML 구성완료)
  • 2. 아이디 찾기 (HTML 구성완료)
  • 3. 비밀번호 찾기 (HTML 구성완료)

  • 비밀번호의 경우는 비밀번호를 초기화 해야하기 때문에 redirectAttributes.addAttribute 속성을 사용하여 해당 회원의 아이디를 전송한 후에, HTML에서 전송하는 Form에 hidden으로 해당 회원의 아이디를 집어넣어서 POST로 데이터 격납시에 해당 회원을 식별할 수 있도록한다.

  • RegisterController (회원등록 프로세스) (5)

  • AboutController (누리컬쳐 소개) (3)

  • div 박스 클릭 시에 해당 링크로 이동하는 이벤트리스너 요소가 많기 때문에, 해당 요소는 JS의 이벤트리스너 요소를 사용해서 div가 동적으로 사용할 수 있는 느낌을 준다. 프로젝트 구현 시간이 남아있다면 hover 속성과 display 속성을 이용해서 커서를 문화누리카드 이미지에 올릴 경우, 링크로 이동한다는 메세지가 나타나게 만들어준다.

  • 1. 누리컬쳐 서비스 설명 페이지 (HTML 구성완료)
  • 2. 누리컬쳐와 문화누리카드 페이지 (HTML 구성완료)

  • 가맹점 설명 페이지에서는, 기존의 정적인 요소를 사용하는 것이 아닌, 등록되어 있는 가맹점을 list 형태로 출력한다. 템플릿의 table 요소를 사용하여 간략하게 등록되어 있는 가맹점의 정보와 해당 가맹점의 URL을 출력시킨다. URL에 해당 가맹점의 정보를 포함하여 GET 요청을 하는 경우, 해당 가맹점의 상세 페이지가 출력된다.
